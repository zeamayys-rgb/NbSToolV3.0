Executive summary

Six QA waves carried the NbS Tool from broken primary flows and an inconsistent palette to a functional, token-driven, mobile-ready, WCAG 2.1 AA-aligned and performance-tuned product. Each wave followed the same protocol — audit and report first, remediate on approval, then verify. Waves 1–4 were run and verified directly; Waves 5–6 (performance and code quality) were executed in a separate session and have since been independently re-verified against the source — all 22 fixes confirmed present, with two follow-up corrections and several calibration notes captured in the Verification section. One item — heading hierarchy (A5) — remains intentionally held for a maintainer decision.
