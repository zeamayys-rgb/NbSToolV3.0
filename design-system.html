ASEAN NbS Platform · Design-to-MRV

A calm, spatial system for Nature-based Solutions.

White-surfaced, neutral-gray UI anchored by a teal-green brand, with full semantic ramps, light & dark themes, Acumin Pro Condensed for display and Poppins for UI — on a 4px spacing system, rounded shape language, and a layered shadow ladder for depth.

Color

Two layers: a primitive palette (the only raw hexes that exist) and semantic system tokens that reference it and flip with theme. Reference system tokens for anything theme-aware; reach for a primitive step only when you need a specific tint.

03

Spacing & Layout

One 4px-based scale governs every gap and pad. Snap stray values to the nearest step — don't reproduce the organic drift of the raw file.

Common V3 layout rhythms to keep for parity: section padding 100/24/0/24, content padding 32/0, footer padding 64/80, panel gap 20, main-content gap 32, card gap 8–16.
04

Border Radius

A curated radius set. Default controls use md (6px); cards and panels use lg (8px); modals use xl (12px).

05

Elevation

A six-step shadow ladder — pick by intent, not by scraping. Layered, low-opacity shadows; never flatten to a single hard shadow.

Derive every state from the system tokens — step fills one shade darker on hover, add a 3px low-opacity ring on focus. See the Components tab for these states applied live per component.

07

Rules

The short version. The full reference lives in DESIGN.md at the repo root.

Do

  • Use the primary family for brand actions; reserve success / danger / warning / purple for their semantics.
  • Pair each *-main color with its *-second tint for subtle backgrounds.
  • Acumin Pro Condensed for display/headlines, Poppins for body/UI.
  • Snap spacing to the 4px scale; default controls radius-md, cards radius-lg.
  • Honor both light and dark values when building themed components.
  • Compose from the Components tab first; promote any pattern used 3+ times.

Don't

  • Introduce hexes outside the palette — use the nearest primitive step or adjust opacity.
  • Repurpose semantic colors (e.g. danger for non-error emphasis).
  • Bring back legacy fonts (Roboto, Lato, DM Sans, IBM Plex, SF Pro, Inter for body).
  • Invent spacing or radii outside the scales — add to DESIGN.md first if genuinely needed.
  • Flatten the shadow system into single hard shadows.
  • Fork a new variant of an existing component — extend the documented one instead.
